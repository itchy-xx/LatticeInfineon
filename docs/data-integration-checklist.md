# Real-data integration checklist
- Confirm approved storage, read-only access, owner and authoritative source.
- Profile format, encoding, keys, nulls, units, timestamps and cadence.
- Define freshness, confidence, conflict, retention and deletion rules.
- Add versioned adapters/mappings without leaking raw names into domain code.
- Add synthetic/anonymized regression fixtures and quality tests.
- Review export control, residency, PII and partner constraints.
- Update contracts, migrations, repositories, APIs and UI labels.
